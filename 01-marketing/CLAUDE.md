# CLAUDE.md — 마케팅 캠페인 성과 대시보드

## 이 프로젝트의 목표
`data/`의 캠페인 CSV를 받아, **이 폴더의 디자인을 그대로 따르는** 채널별 성과 대시보드 HTML을 `output/`에 만든다.
"예쁜 화면"이 아니라 "3초 안에 어떤 채널에 예산을 더 넣을지 보이는 화면"을 만든다.

---

## 가장 중요한 규칙 (반드시 먼저 읽을 것)
**처음부터 새로 디자인하지 않는다.** 항상 `templates/reference.html`을 먼저 Read 하고,
그 **구조·레이아웃·색상·차트 종류·카드 패턴을 그대로 재사용**한 뒤, **숫자와 인사이트만 새 데이터로 교체**한다.
- 레퍼런스는 "품질 기준선"이다. 결과물이 레퍼런스보다 단순하거나 다른 톤이면 실패다.
- 단, **복붙이 아니다.** 데이터는 반드시 새로 분석해서 실제 계산값을 넣는다 (아래 KPI 정의대로).

---

## 디자인 토큰 (reference.html과 동일하게 고정)
```css
--bg:#f3f5f5;        /* 컨텐츠 배경 */
--card:#ffffff;      /* 카드 배경 */
--border:#e8e8e8;    /* 카드 테두리 */
--sidebar:#000000;   /* 사이드바 */
--ink:#0a0a0a;       /* KPI 수치 — 검정 고정, 빨강/초록 금지 */
--green:#35c995;     /* 좋음/상승 */
--red:#ef7d86;       /* 나쁨/하락 */
/* 채널 팔레트 (진→연, 성과순) */
--c1:#2E4A8F; --c2:#4E6FB5; --c3:#7A9BD4; --c4:#A9C3E8; --c5:#C5D9F0;
```
- 폰트: **Inter** (reference와 동일)
- 차트: reference가 쓰는 방식 그대로 — `chart.js`(CDN) + `<canvas>`, 아이콘은 인라인 `<svg>`
- KPI 수치 색은 **검정 고정**. 빨강/초록은 증감 화살표·뱃지에만.

---

## 핵심 KPI 정의 (계산식 고정)
| 지표 | 계산식 | 표기 |
|---|---|---|
| ROAS | revenue / spend | `6.15x` (배수) |
| ROI | (revenue − spend) / spend × 100 | `%` |
| CTR | clicks / impressions × 100 | `%` |
| CVR | conversions / clicks × 100 | `%` |
| CPA | spend / conversions | `원` |
| CPC | spend / clicks | `원` |
- 비율은 `%`, 증감은 `%p`로 구분 표기. 절대값/상대값 혼용 금지.
- 채널 순위는 **ROAS 기준** 내림차순. 1위 채널은 `--c1`, 꼴찌는 `--c5`.

## 대시보드 구성 (reference의 섹션 순서 유지)
1. **상단 KPI 카드 4개**: 전체 ROAS · 총 광고비 · 총 전환수 · 최고 채널(이름+ROAS)
   - 카드 제목은 질문/결론형 ("어떤 채널이 돈을 벌어주나?")
2. **채널별 성과 비교** — 정렬된 가로 막대 (ROAS), 1위만 진한 `--c1`
3. **ROAS 시계열 추이** — 라인, 채널별 색상
4. **클러스터/효율 분석** — scatter (x=CVR, y=ROAS, 버블=spend)
5. **예산 배분 권고 표** — 채널 / 현재 비중 / 권고 비중 / 근거 한 줄

## 금지
- reference를 무시하고 처음부터 새 레이아웃을 만드는 것
- KPI 수치에 색 입히기 / 한쪽 변만 있는 border(border-left 등 단독)
- 차트 종류를 이유 없이 늘리기 / 장식용 그라디언트
- 데이터 분석 없이 레퍼런스 숫자를 그대로 남겨두는 것
