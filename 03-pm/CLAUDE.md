# CLAUDE.md — 프로덕트(PM) 지표 위계 대시보드

## 이 프로젝트의 목표
`data/`의 프로덕트 지표 CSV를 받아, **이 폴더의 디자인을 그대로 따르는** 지표 위계·원인 분석 대시보드 HTML을 `output/`에 만든다.
"예쁜 화면"이 아니라 "3초 안에 어떤 Input 지표를 먼저 고쳐야 할지 보이는 화면"을 만든다.

---

## 가장 중요한 규칙 (반드시 먼저 읽을 것)
**처음부터 새로 디자인하지 않는다.** 항상 `templates/reference.html`을 먼저 Read 하고,
그 **구조·레이아웃·색상·차트 종류·카드 패턴을 그대로 재사용**한 뒤, **숫자와 인사이트만 새 데이터로 교체**한다.
- 레퍼런스는 "품질 기준선"이다. 결과물이 레퍼런스보다 단순하거나 다른 톤이면 실패다.
- 단, **복붙이 아니다.** 데이터는 반드시 새로 분석해서 실제 계산값을 넣는다 (아래 지표 정의대로).
- const 배열(`treeData`/`frustData`/`timingData`/`planData`/`rfmData`/`causeData`/`metricsTable`)의 **필드명·형식은 100% 그대로** 두고 값만 교체한다. 형식이 깨지면 차트가 깨진다.

## 지표 위계 (North Star → Primary → Input → Guardrail)
PM 대시보드의 뼈대는 단일 KPI가 아니라 **위계 트리**다. (`treeData`가 이를 구동)

| 계층 | 의미 | 예시 |
|---|---|---|
| **North Star** | 사업이 향하는 단일 결과 지표 | 구독 매출 |
| **Primary** | North Star를 직접 움직이는 중간 지표 | 구독자 수 |
| **Input** | 팀이 직접 통제 가능한 선행 지표 | 재결제율 · 구독/체험 전환율 · 액티브 리텐션 |
| **Guardrail** | 성장을 위해 망가지면 안 되는 방어 지표 | 좌절 경험률 · 구독자 증감 비율 |

## 지표 정의 (계산식·분모·제외조건 고정)
| 지표 | 계산식 | 분모 / 제외조건 | 표기 |
|---|---|---|---|
| 재결제율 | 1 − 구독 해지율 | 분모=직전 결제 구독자, 환불·체험 제외 | `%` |
| 구독 전환율 | 결제 전환 / 무료체험 완료자 | 분모=체험 완료자(미완료 제외) | `%` |
| 체험 전환율 | 무료체험 시작 / 핵심기능 미리보기 노출 | 분모=노출자 | `%` |
| 액티브 리텐션 | 핵심기능 사용자 / W−1 구독자 | 분모=직전 주 구독자 | `%` |
| 좌절 경험률 | 핵심기능 없는 문제·검색 / 전체 검색 | 분모=전체 검색(봇 제외) | `%` |
| 구독자 증감 비율 | (신규 + 복귀) / 이탈 | `< 1`이면 순감(위험) | `0.82` (배수) |
- 비율은 `%`, 증감은 `%p`로 구분 표기. 절대값(`-88명`)/상대값(`-3.2%p`)을 한 카드 안에서 혼용하지 않는다.
- 추정 매출 위험(`~₩9M/주`)은 **"~" 또는 "추정"** 문구를 함께 쓴다.

## 원인 분석 / 세그먼트 차원 (`causeData`·`rfmData`·`timingData`·`planData` 구동)
- 이탈 사유는 **설문 + 행동 데이터 교차 검증**으로 `verified`(검증됨/요검증)를 표시한다. 설문만 있는 사유는 `요검증`.
- 세그먼트(RFM): 가입기간·결제횟수·마지막 로그인 기준으로 6분류. `이탈 위험` 비중이 가장 높으면 최우선 개입.
- 이탈 시점(가입 후 몇 주차)·플랜별 분포로 "누가/언제" 이탈하는지 분해한다.

## 대시보드 구성 (reference의 STEP 순서 유지)
1. **STEP 1 — 상단 KPI 카드 6개**: 구독자 증감 비율·재결제율·체험 전환율·구독 전환율·좌절 경험률·액티브 리텐션 (각 스파크라인+정의)
   - 카드 제목 영역은 결론/질문형 step-title ("지금 어디가 문제인가?")
2. **STEP 2 — 지표 구조**: Metric Hierarchy 트리(`treeData`) + 구독자 증감 비율 Pulse + 코호트 리텐션 곡선
3. **STEP 3 — 이탈 분석**: RFM 세그먼트 트리맵(`rfmData`) + 이탈 트렌드/특성(`frustData`·`timingData`·`planData`) + 이탈 사유 TOP5(`causeData`)
4. **STEP 4 — 액션 플랜**: AI 인사이트(우선순위) + 계층별 Input Metric 상태 테이블(`metricsTable`)

## Action 도출 원칙
- 위계를 거슬러 올라가며 **가장 상류의 망가진 Input 지표**를 먼저 고친다 (신규 유입 확대보다 재결제율 우선).
- 각 액션에는 **가설 · 검증 방법 · 예상 효과**를 붙인다. A/B 불가 시 시계열 추적 명시.

## 금지
- reference를 무시하고 처음부터 새 레이아웃을 만드는 것
- KPI 수치에 색 입히기(증감비율 같은 즉각 위험 지표 제외) / 한쪽 변만 있는 border(border-left 등 단독)
- 차트 종류를 이유 없이 늘리기 / 장식용 그라디언트
- 데이터 분석 없이 레퍼런스 숫자를 그대로 남겨두는 것
- const 배열 필드명/형식을 바꾸는 것 (차트 깨짐)
- 검증 안 된 사유(`요검증`)를 검증된 것처럼 단정하는 것
