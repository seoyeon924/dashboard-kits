# CLAUDE.md — HR 채용 퍼널 & People Analytics 대시보드

## 이 프로젝트의 목표
`data/`의 채용 CSV를 받아, **이 폴더의 디자인을 그대로 따르는** 채용 퍼널·이탈 리스크 대시보드 HTML을 `output/`에 만든다.
"예쁜 화면"이 아니라 "3초 안에 채용 퍼널의 어느 단계가 병목이고 누구를 먼저 붙잡아야 하는지 보이는 화면"을 만든다.

---

## 가장 중요한 규칙 (반드시 먼저 읽을 것)
**처음부터 새로 디자인하지 않는다.** 항상 `templates/reference.html`을 먼저 Read 하고,
그 **구조·레이아웃·색상·차트 종류·카드 패턴을 그대로 재사용**한 뒤, **숫자와 인사이트만 새 데이터로 교체**한다.
- 레퍼런스는 "품질 기준선"이다(제목 "HR People Analytics OS", 778줄). 결과물이 레퍼런스보다 단순하거나 다른 톤이면 실패다.
- 단, **복붙이 아니다.** 데이터는 반드시 새로 분석해서 실제 계산값을 넣는다 (아래 KPI 정의대로).
- 레퍼런스의 `const` 데이터 배열만 교체한다. **JS 로직·차트 옵션·배열 필드명/형식은 100% 그대로 둔다** (안 그러면 차트가 깨진다).

## 디자인 토큰 (reference.html :root에서 직접 추출 — 동일하게 고정)
```css
--bg:#f5f5f3;        /* 컨텐츠 배경 */
--card:#ffffff;      /* 카드 배경 */
--border:#e8e8e8;    /* 카드 테두리 */
--sidebar:#000000;   /* 사이드바 */
--t1:#1a1a1a;        /* KPI 수치 — 검정 고정 */
--t2:#2f2a25; --t3:#888; --t4:#aaa;   /* 본문/보조 텍스트 */
/* 성과순 팔레트 (진→연) */
--c1:#57b497; --c2:#2a2a2a; --c3:#7abda0; --c4:#a9d4c3; --c5:rgba(87,180,151,0.16);
--green:#35c995;     /* 좋음/안정 */
--green2:#57b497;
--amber:#f59e0b;     /* 주의/경고 */
--red:#ef7d86;       /* 위험/나쁨 */
--font:'Inter';  --r:12px;
```
- 폰트: **Inter** (reference와 동일). 차트 라벨 폰트는 reference가 쓰는 그대로.
- 차트: reference 방식 그대로 — `chart.js@4.4.0`(CDN) + `<canvas>`, 스파크라인은 직접 canvas 드로잉, 아이콘은 인라인 `<svg>`.
- KPI 수치 색은 **검정(--t1) 고정**. 빨강/초록/주황은 위험도 강조·증감 뱃지·이탈 위험 행에만.

## 핵심 KPI 정의 (계산식 고정)
| 지표 | 계산식 | 표기 |
|---|---|---|
| 단계별 통과율 | passed / candidates × 100 | `%` (단계마다) |
| 퍼널 전환율(전체) | 입사 candidates / 지원 candidates × 100 | `%` |
| 단계 이탈률 | (1 − 통과율) × 100 | `%` |
| time-to-hire | 지원→입사 단계 avg_days 누적(부서별 평균) | `일` |
| offer acceptance rate | offer_accepted / offer_sent × 100 | `%` |
| 부서별 채용 현황 | 부서별 입사 인원, 부서별 전체 전환율 | `명` / `%` |
| 이탈 리스크 점수 | 근속·성과·몰입도·마지막 1:1 기반 AI 예측 점수(0~100) | `%` |
- 비율은 `%`, 증감은 `%p`로 구분 표기. 절대값/상대값 혼용 금지.
- 부서 순위는 **자발적 이직률 / 전환율 기준**. 위험 부서일수록 빨강(--red)→주황(--amber)→회색(#ccc) 순.
- 퍼널 단계 순서는 **지원 → 서류 → 1차면접 → 2차면접 → 오퍼 → 입사** 고정.

## 대시보드 구성 (reference의 STEP·섹션 순서 유지)
1. **STEP 1 — 상단 KPI 6개**: 총 재직 인원 · 신규 입사자 · 자발적 퇴사자 · 고위험 이탈 인원 · 평균 채용 소요일 · 자발적 이직률
   - 카드 제목은 질문/결론형. 각 카드에 스파크라인(spark0~5).
2. **STEP 2 — 인원 변동 + 위험 등급**: 부서별 채용 소요일 추이(line) · Q1 인원 변동(워터폴) · 이직 위험도 등급 분포
3. **STEP 3 — 드릴다운**: 부서별 자발적 이직률(가로 막대) · **Q1 채용 퍼널(단계별 전환율)** · 성과 등급 분포(bar)
4. **STEP 4 — 개인별 탐지**: AI People 알림 & 액션(aiItems) · 이탈 위험 인원 테이블(riskData)
