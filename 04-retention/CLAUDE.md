# CLAUDE.md — 코호트 리텐션 & LTV 대시보드

## 이 프로젝트의 목표
`data/`의 구독 CSV를 받아, **이 폴더의 디자인을 그대로 따르는** 코호트 리텐션·MRR·LTV 대시보드 HTML을 `output/`에 만든다.
"예쁜 화면"이 아니라 "3초 안에 어떤 코호트가 새고 있고, 어디에 리텐션 자원을 넣을지 보이는 화면"을 만든다.

---

## 가장 중요한 규칙 (반드시 먼저 읽을 것)
**처음부터 새로 디자인하지 않는다.** 항상 `templates/reference.html`을 먼저 Read 하고,
그 **구조·레이아웃·색상·차트 종류·카드 패턴을 그대로 재사용**한 뒤, **데이터 배열과 인사이트만 새 데이터로 교체**한다.
- 레퍼런스는 "품질 기준선"이다. 결과물이 레퍼런스보다 단순하거나 다른 톤이면 실패다.
- 단, **복붙이 아니다.** 데이터는 반드시 새로 분석해서 실제 계산값을 넣는다 (아래 KPI 정의대로).
- **JS 로직·차트 초기화·배열 필드명/형식은 절대 바꾸지 않는다.** const 데이터 값만 교체한다. (필드명이 바뀌면 차트가 깨진다.)

## 디자인 토큰 (reference.html `:root`에서 직접 추출 — 동일하게 고정)
```css
--bg:#F0F2F7;        /* 컨텐츠 배경 */
--card:#FFFFFF;      /* 카드 배경 */
--border:#E2E8F0;    /* 카드 테두리 */
--sidebar:#2E3B55;   /* 사이드바 (네이비) */
--t1:#0F172A;        /* KPI 수치 — 검정 고정, 빨강/초록 금지 */
--t2:#334155; --t3:#64748B; --t4:#94A3B8;  /* 보조 텍스트 위계 */
--green:#10B981;     /* 좋음/상승/안전 */
--red:#EF4444;       /* 나쁨/하락/위험 */
--green-bg:#ECFDF5; --red-bg:#FEF2F2;       /* 뱃지 배경 */
/* 코호트/플랜 팔레트 (진→연, 리텐션 높은 순) */
--c1:#2A7DB5; --c2:#2E3B55; --c3:#4A7AB5; --c4:#85B8D8; --c5:#C5DCF0;
--r:10px;            /* 카드 radius */
```
- 폰트: **Pretendard** (reference와 동일, CDN)
- 차트: reference가 쓰는 방식 그대로 — `chart.js@4.4.0`(CDN) + `<canvas>`, KPI 스파크라인은 직접 Canvas 드로잉, 아이콘은 인라인 `<svg>`
- KPI 수치 색은 **검정(`--t1`) 고정**. 빨강/초록은 증감 화살표·뱃지·위험 행에만.
- 코호트 히트맵 색: `chColor()` 규칙대로 — 리텐션 ≥80% `--c2`, ≥70% `--c1`, ≥60% `--c3`, ≥50% `--c4`, 그 미만 `--c5`.

## 핵심 KPI 정의 (계산식 고정)
| 지표 | 계산식 | 표기 |
|---|---|---|
| 코호트 리텐션율 | 해당 코호트의 M{n} 활성 유저 / M0 유저 × 100 | `%` |
| MRR | 해당 월 활성 유저 mrr 합계 | `₩…M` |
| 신규 MRR | 그 달에 처음 가입한(signup_month==active_month) 유저 mrr 합 | `₩…M` |
| Expansion MRR | 동일 유저 전월 대비 mrr 증가분 합 | `₩…M` |
| Churned MRR | 그 달에 churned=1 된 유저의 직전 mrr 합 (음수 표기) | `−₩…M` |
| Revenue Churn rate | 이탈 MRR / 전월 총 MRR × 100 | `%` |
| NRR | (기초MRR − churned + expansion) / 기초MRR × 100 | `%` |
| 평균 LTV | 유저별 생애 mrr 합의 전체 평균 (= ARPA / churn 근사) | `₩…K/M` |
| Payback(참고) | CAC / 월 ARPA (CAC 미제공 시 표기 생략) | `개월` |

- 비율은 `%`, 증감은 `%p`로 구분 표기. 절대값/상대값/누적값 혼용 금지.
- 코호트는 **가입월(signup_month) 오름차순** 행, **경과월(M0..M5)** 열. M0는 항상 100%.
- 아직 관측 안 된 (가입월 + 경과월 > 데이터 마지막 월) 셀은 `null` 유지 → 회색 빈 칸으로 렌더.
- 플랜 세그먼트는 churn 낮은(리텐션 좋은) 순으로 색 `--c1`~`--c5` 부여.

## What-if (가정 변경 시 영향) 관점
- 대시보드는 슬라이더로 **churn / expansion / 신규 MRR** 가정을 바꿔 예측 MRR·NRR·3개월 포워드를 보여준다.
- `BASE_MRR`(현재 총 MRR), `BASE_NRR`(현재 NRR)을 실제 계산값으로 고정하면 What-if 로직이 그 위에서 돈다.
- 권고는 항상 "이 가정을 X로 바꾸면 MRR이 Y만큼 움직인다" 형태로 — 막연한 "좋아진다" 금지.

## 대시보드 구성 (reference의 섹션 순서 유지)
1. **상단 KPI 카드 6개**: MRR · M6 코호트 리텐션 · 평균 LTV · Revenue Churn · 신규 MRR · NRR (각 스파크라인 포함)
   - 카드 제목/정의는 결론·기준형 ("이탈 MRR / 전월 MRR · 매출 기준")
2. **MRR 구성 추이** — 신규/확장/이탈 스택 막대 + 성장률 라인 (월별)
3. **What-if 시뮬레이터** — churn/expansion/신규 슬라이더 → 예측 MRR·NRR
4. **코호트 리텐션 히트맵** — 가입월 × 경과월 매트릭스 (chColor 색 규칙)
5. **이탈 사유 Pareto** — 건수 기준 / 매출 기준 토글
6. **LTV 분포** — 매출 구간별 고객 비중 히스토그램
7. **AI 인사이트 카드** — 이탈 위험·코호트 하락·매출 영향 순위 (결론+권장 액션)
8. **플랜 세그먼트 표** — 고객수·MRR·Churn·NRR·LTV·MoM·권장 액션 배지

## 금지
- reference를 무시하고 처음부터 새 레이아웃을 만드는 것
- const 배열의 **필드명·길이·형식을 바꾸는 것** (차트 깨짐) — 값만 교체
- KPI 수치에 색 입히기 / 한쪽 변만 있는 border(border-left 등 단독)
- 차트 종류를 이유 없이 늘리기 / 장식용 그라디언트
- 데이터 분석 없이 레퍼런스 숫자를 그대로 남겨두는 것
- 코호트 M0를 100% 아닌 값으로 두거나, 미관측 셀을 0으로 채우는 것
